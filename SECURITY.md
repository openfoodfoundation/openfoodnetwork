# Security Policy

## Supported Versions

The latest release will be supported with security updates. 
See the [releases](https://github.com/openfoodfoundation/openfoodnetwork/releases) page to check the latest release.

## Reporting a Vulnerability

If you discover a security vulnerability, we'd appreciate a non-public disclosure. 
Developers can be contacted privately by creating a new [**Security Advisory on GitHub**](https://github.com/openfoodfoundation/openfoodnetwork/security/advisories/new).

## Requirements for a Valid Report
- Please ensure the issue is reproducible on main.
- Please ensure a fully working, end-to-end reproducer is provided.
- Please ensure the reproducer is real-world and not simulated or abstracted.
- Please ensure the reproducer demonstrably violates a security boundary.
- We will try to triage and fix issues in a timely manner, but we cannot guarantee any fixed timeline for issue resolution.

Thank you for your help in maintaining this open source project for the benefit of the community!
